项目经历

  • 基于Node.js并发微博数据系统的设计和实现
    该项目的主要工作是设计和实现了基于Node.js的高并发微博数据系统，采用了基于Oauth2.0认证的新浪API作为数据的来源。基本的功能包括但不限于用户注册和登陆，获取和更新微博内容，用户信息，和微博信息（评论数，转发数，点赞数，是否为用户原创微博等），生成用户个人数据表等。由于整个系统应用了MVC软件设计模式，本着RESTful API 和前后端分离的原则，所有后端经过Nodejs采集到的数据，通过API的方式传递到前端，充分实现高耦合，低内聚。
    通过（node-schedule）的定时器模块被用于定时地获取，储存，并更新数据。此外，前端部分基于AdminLTE，并实现了多个页面，数据通过图表的形式做了较好的可视化展示（技术方案为百度的echart）。
    由于Node.js的单线程工作模式，为了充分利用服务器多核的优势，在服务器端使用了Nginx作为反向代理。
    数据存储： 基于MySQL（Sequelize）实现了数据持久化存储。为了提高响应速度，利用Redis作为数据缓冲层，实现了数据在内存级的存储，减少了系统直接访问MySQL数据库的频率，进一步提高了系统的响应速度，和高并发的适应力。
    由于此项目涉及到多出异步操作，为了加强程序的可读性，可拓展性。采用了TJ大神的Co框架作为异步的解决方案。（当时async await并未被Nodejs所支持所以并未选用）。（此项目为导师项目所以并未上传github）
